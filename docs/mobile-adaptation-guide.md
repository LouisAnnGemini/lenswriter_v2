# 电脑端到移动端改造与适配指南

本文档总结了在将应用从电脑端（Desktop）向移动端（Mobile）进行适配和改造时，需要重点关注的核心差异与设计/开发要点。移动端适配并非简单的“缩小元素”，而是基于设备特性和用户习惯的**信息层级重组**。

## 1. 布局与空间利用 (Layout & Space)

*   **多栏变单栏 / 层层递进：** 手机屏幕宽度有限，电脑端常见的左右多栏布局（如左侧列表+右侧详情）在手机端应改为连续滚动的单栏布局，或者拆分为几个全屏的板块，通过点击进行层层递进的导航。
*   **释放垂直空间（警惕固定元素）：** 手机屏幕的垂直空间非常宝贵。电脑端常驻的固定 Banner、厚重的 Header 或 Footer，在手机端会造成严重的视觉挤压效果。应尽量精简固定元素，或者在向下滚动时自动隐藏顶部导航栏。

## 2. 交互方式的根本转变 (Hover vs. Touch)

*   **悬停状态（Hover）的消失：** 移动端没有鼠标悬停的概念。电脑端依赖 Hover 显示的提示（Tooltip）、隐藏操作按钮（如编辑、删除）或下拉菜单，必须转化为明确的点击事件（如点击“...”呼出操作菜单），或者直接常驻显示在界面上。
*   **触控区域（Touch Targets）：** 手指触控的精准度远低于鼠标。所有可点击元素（按钮、链接、选项）必须足够大（建议至少 44x44 像素），并且元素之间需要留有更大的安全间距，以防止误触。

## 3. 导航结构的重组 (Navigation)

*   **导航栏的降维：** 电脑端宽阔的屏幕适合左侧固定侧边栏或复杂的顶部导航。在手机端，应将侧边栏收纳进汉堡菜单（Hamburger Menu），或者针对最核心的功能使用**底部标签栏（Bottom Tab Bar）**，因为屏幕底部是单手拇指最容易触达的“舒适区”。
*   **渐进式展现与信息降噪：** 手机端讲究“聚焦当前，层层递进”。次要信息和低频操作必须被折叠或隐藏，保持界面的清爽，避免给用户带来信息过载的压迫感。

## 4. 弹窗与浮层的呈现 (Modals vs. Bottom Sheets)

*   **摒弃居中弹窗：** 电脑端经常使用屏幕居中的弹窗（Modal）。但在手机端，居中弹窗不仅显得局促，而且关闭按钮（通常在右上角）很难用单手拇指够到。
*   **拥抱底部抽屉：** 手机端更适合使用从底部弹出的抽屉（Bottom Sheet/Drawer），或者直接跳转到一个新的全屏页面。底部抽屉符合单手操作习惯，且可以通过向下滑动的手势轻松关闭。

## 5. 软键盘遮挡问题 (Virtual Keyboard Occlusion)

*   **可视区域骤减：** 手机端一旦点击输入框调出软键盘，屏幕可视高度会瞬间减少将近一半。
*   **焦点与操作跟随：** 如果输入框位于屏幕中下方，极易被键盘遮挡。设计与开发时需确保焦点元素能自动滚动到可视区域内；同时，底部的固定操作按钮（如“保存”、“提交”）在键盘弹起时，需要跟着浮动到键盘上方，确保用户随时可以完成操作。

## 6. 数据密集型组件的降维 (Data-heavy Components)

*   **表格（Tables）转卡片：** 电脑端可以轻松展示多列数据表格。手机端通常需要将表格“降维”成**卡片列表（Card List）**，每张卡片纵向排列原本在同一行的数据。
*   **看板（Kanban/Board）转单列：** 类似 Trello 的多列看板在手机端无法平铺，通常需要变成单列展示，通过左右滑动屏幕或顶部的 Tab 标签来切换不同的列（状态）。

# Metro (故事地铁线) 功能需求方案 (V2 完整版)

## 一、 功能定位与基础设置
1. **全局标签页**：新增一个名为 `Metro` 的顶级标签页（与 Writing, Notes, Montage, Timeline 等并列）。
2. **自定义配置**：在 Customize Tabs 设置中加入 Metro，允许用户在 Design、Review、Management 三个模式下独立设置它的显示/隐藏和排序位置。

## 二、 核心数据逻辑
1. **多线路管理**：
   * 用户可以创建、重命名、删除多条“线路（Line）”（例如：主角复仇线、反派阴谋线、感情支线等）。
   * 每条线路拥有独立的视图和数据结构。
2. **节点与事件卡片（Event Card）的解耦**：
   * 线路上的每一个**节点（Node）**本质上是对某个**事件卡片（Event Card）**的引用。
   * **复用性**：同一个事件卡片可以被不同的线路引用（例如“男女主相遇”这个事件，既在男主线里，也在女主线里）。
3. **初始状态**：
   * 新建一条线路时，系统会自动在画布上生成一个初始的主线节点，并在后台同步新建一个对应的 Event Card。

## 三、 界面与可视化表现
1. **线路管理区**：界面提供区域（如侧边栏或顶部下拉菜单）用于切换、新增、管理不同的故事线路。
2. **地铁图画布区**：
   * 一个支持拖拽平移、缩放的无限画布。
   * **节点外观**：显示为地铁站一样的圆点（可根据 Event Card 颜色显示），节点旁边显示 Event Card 的标题。
3. **🚇 核心排版与坐标规则**：
   * **主线/同级推进**：默认从左到右水平排列。假设相邻两个节点 A 和 B 的标准水平间距为 `1`（例如 200px）。
   * **支线生成位置**：当从节点 A 衍生出支线时，支线第一个节点的位置将严格遵循 `(Ax + 0.5, Ay ± 0.5)` 的规则。即：在水平方向上位于 A 和 B 的正中间，垂直方向上偏移半个标准间距，形成完美的 45 度角连线。
   * **支线后续推进**：支线节点本身可以继续“后置”节点（继续向右水平推进），也可以继续长出“支线的支线”（无限层级树状发展，继续遵循 `+0.5, ±0.5` 规则）。

## 四、 核心交互逻辑
1. **双击节点**：直接弹出该节点对应的 Event Card 详情编辑面板。
2. **单击节点**：弹出一个悬浮菜单（Popover），包含以下 5 个核心操作：
   * ➕ **前置 (Insert Before)**：在当前节点的**左侧**插入一个新节点，并自动创建一个空的 Event Card。原先连接在当前节点左侧的线会连到新节点上。
   * ➕ **后置 (Insert After)**：在当前节点的**右侧**插入一个新节点，并自动创建一个空的 Event Card。原先连接在当前节点右侧的线会连到新节点上。
   * 🔄 **替换 (Replace)**：弹出一个带搜索框的下拉列表，允许用户通过标题搜索现有的 Event Card。选中后，当前节点引用的 Event Card 被替换，但节点在地铁图上的位置和拓扑结构不变。
   * 🌿 **支线 (Branch)**：在当前节点的右上方或右下方（45度角，遵循上述坐标规则）创建一个新节点，并自动创建一个空的 Event Card。
   * 🗑️ **删除 (Remove)**：
     * 将当前节点从这条地铁线上移除。
     * **级联移除**：如果该节点包含支线，其下属的所有支线节点将**连同该节点一起从画布上移除**。
     * **数据安全**：此操作**仅移除画布上的节点引用和连线，绝对不会删除后台的 Event Card 数据**。
     * **线路缝合**：如果删除的是主线/某条线中间的节点，删除后，该节点前后的节点会自动连接起来，保证线路不断裂。

## 五、 技术架构设计

### 1. 数据模型定义 (Types)
```typescript
// 支线的方向：1 代表向右下 (Ay + 0.5)，-1 代表向右上 (Ay - 0.5)
export type MetroBranchDirection = 1 | -1;

export type MetroNode = {
  id: string;
  lineId: string;           // 所属的线路 ID
  eventId: string;          // 关联的 TimelineEvent ID
  nextId: string | null;    // 主线/当前线上的下一个节点 ID
  branches: {
    nodeId: string;         // 支线第一个节点的 ID
    direction: MetroBranchDirection; // 支线延伸的方向
  }[];
};

export type MetroLine = {
  id: string;
  workId: string;           // 所属的作品 ID
  title: string;            // 线路名称
  rootNodeId: string | null; // 线路的起点节点 ID
};
```

### 2. 状态管理 (Zustand Slice)
创建 `metroSlice.ts`，在全局状态中增加：
* `metroLines: MetroLine[]`
* `metroNodes: MetroNode[]`

### 3. 坐标计算引擎 (Coordinate Engine)
不写死 X 和 Y 的绝对坐标，在组件渲染时通过递归函数动态计算：
1. 找到 `rootNodeId`，将其坐标设为 `(0, 0)`。
2. 沿着 `nextId` 遍历：下一个节点的坐标为 `(当前X + 1, 当前Y)`。
3. 沿着 `branches` 遍历：支线第一个节点的坐标为 `(当前X + 0.5, 当前Y + direction * 0.5)`。
4. 支线节点继续沿着它的 `nextId` 遍历：坐标为 `(当前X + 1, 当前Y)`。

### 4. 核心操作逻辑 (Actions)
* **前置 (Insert Before)**：创建新节点 N。找到当前节点 C 的父节点 P。将 P 的指针指向 N，将 N 的 `nextId` 指向 C。
* **后置 (Insert After)**：创建新节点 N。将 N 的 `nextId` 指向 C 的 `nextId`。将 C 的 `nextId` 指向 N。
* **支线 (Branch)**：创建新节点 N。在 C 的 `branches` 数组中推入 `{ nodeId: N.id, direction }`。
* **删除 (Remove)**：找到 C 的父节点 P。将 P 的指针直接指向 C 的 `nextId`（完成缝合）。递归收集 C 及其所有 `branches` 下的节点 ID，从 `metroNodes` 数组中批量删除（完成级联删除）。

### 5. 渲染层设计 (UI)
* 使用 `react-zoom-pan-pinch` 提供无限画布的拖拽和缩放体验。
* 使用 SVG 绘制节点之间的连线。
* 节点本身使用绝对定位的 HTML 元素，通过计算出的 `(X * 200px, Y * 100px)` 映射到屏幕像素坐标。

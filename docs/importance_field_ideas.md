# Timeline Event "Importance" 字段改造方案备忘录

**背景说明：**
目前 `TimelineEvent` 中的 `importance` 字段（1-5）尚未被充分利用。未来的改造方向希望侧重于**分析和管理能力**的加强。但考虑到在故事构思初期，过早地强迫作者分清所有线路会增加认知负担，因此暂时搁置改造，将方案记录于此，待后续时机成熟时再行评估和开发。

---

## 方案一：语义化颜色与标签（Semantic Color Coding）
*   **概念**：废弃或整合现有的自由 `color` 字段，将 `importance` 严格映射为“事件级别”，并赋予固定的颜色和标签。
*   **视觉表现**：
    *   **1 - 白色/灰色**：背景设定 / 日常过渡 (Background)
    *   **2 - 蓝色**：角色发展 / 支线情节 (Character/Subplot)
    *   **3 - 绿色**：正常剧情推进 (Plot Progression)
    *   **4 - 黄色**：重要转折 / 揭秘 (Turning Point)
    *   **5 - 红色**：高潮 / 核心冲突 (Climax)
*   **核心价值**：规范化视觉语言，用户不需要每次都纠结选什么颜色，一看颜色就知道这个事件在故事中的分量。

## 方案二：缩放级别与过滤器（Zoom Levels / LOD）—— *偏向管理能力*
*   **概念**：将 `importance` 作为信息过滤的层级。
*   **视觉表现**：在界面顶部增加一个“细节滑块（Level of Detail）”或过滤按钮。
    *   当滑块在最高层级时，只显示 `importance = 4 或 5` 的事件（故事骨架/大纲）。
    *   向下滑动时，逐渐显示 3、2 的事件。
    *   最底层显示所有细节（包括 `importance = 1` 的琐碎事件）。
*   **核心价值**：当用户的事件库达到几百个时，时间线会非常拥挤。这个功能可以帮助他们一键“只看主线”或“查看所有细节”，有效解决后期信息过载的问题。

## 方案三：情绪/冲突曲线（Tension Curve / Narrative Arc）—— *偏向分析能力*
*   **概念**：将 `importance` 视为该事件的“冲突强度”或“剧情张力”。
*   **视觉表现**：在 Timeline 或 Montage 界面的顶部/背景，根据每个事件的 `importance` 值（1-5）自动生成一条**起伏的折线图（Sparkline）**。
*   **核心价值**：帮助创作者直观地看到故事的“起承转合”。如果某几章全是 1，说明节奏可能太拖沓；如果全是 5，说明读者可能会审美疲劳。这是一个非常高级且实用的写作辅助功能。

## 方案四：视觉权重排版（Visual Hierarchy）
*   **概念**：不改变颜色，而是改变 UI 元素的**大小和排版**。
*   **视觉表现**：
    *   `importance = 5` 的事件卡片更大、字体更粗、带有明显的边框或阴影（甚至占据整行）。
    *   `importance = 1` 的事件卡片非常迷你，文字颜色较淡，甚至默认折叠。
*   **核心价值**：在 Montage 面板中，让“大事件”像电影海报一样醒目，让“小事件”像便利贴一样紧凑，打破千篇一律的网格布局。

## 方案五：主线与支线（A-Story vs B-Story）
*   **概念**：将这个字段直接改造成 `plotThread`（剧情线级别）。
*   **视觉表现**：5 代表主线（A-Story），3 代表支线（B-Story），1 代表草蛇灰线的伏笔（C-Story）。可以在卡片右上角打上明确的 A/B/C 标签。
*   **核心价值**：直接辅助管理复杂的多线叙事。

---

**未来开发建议：**
根据作者目前的反馈，未来改造时应优先考虑 **方案二（细节过滤）** 和 **方案三（情绪曲线）**，以强化工具的分析与管理属性。
